# Triggers And Outcomes

## Trigger Types

### Failure Triggers

- shell command failure
- test failure
- runtime error
- build failure

### Feedback Triggers

- explicit rejection such as "wrong"
- explicit rejection such as "not right"
- explicit rejection such as "that is not it"
- explicit rejection such as "not what I meant"

### Explicit Review Triggers

- retrospective request
- failure summary request
- memory optimization request
- skill review request

### End-Of-Task Trigger

After every task, check:

- whether a new lesson should be recorded
- whether memory is conflicting or bloated
- whether other skills were used
- whether skill review is needed

## Outcome Selection

### Write To Corrections

Use this for:

- fresh failures
- fresh corrections
- not-yet-stable lessons
- workflow ideas that still need validation

### Promote To Memory

Use this for:

- stable rules
- rules reused across tasks
- long-term requirements clearly stated by the user

### Trigger Skill Review

Use this for:

- used skills that show clear defects
- repeated manual recovery because a skill is incomplete
- inaccurate trigger text that causes misses or false positives

### Compress Memory

Use this for:

- character count above the soft threshold
- heavy duplication
- repeated case notes for the same lesson
